# 概述

## 描述

该度量指标识别计划在 Primavera P6 数据日期启动但没有有效前置逻辑驱动该启动的开放或未启动活动。在一份构建良好的 CPM 进度计划中，活动的启动通常应是因为前置工作、项目通道开放、审批、采购，或其他必要条件在逻辑上已得到满足。

## 测量内容

该度量指标测量启动日期与数据日期对齐、却没有逻辑前置路径解释为何工作可以在该节点开始的活动数量。

## 阈值

目标阈值：0 项活动。

## 重要性

在数据日期启动但没有驱动逻辑的活动可能削弱进度计划的可信度。它们可能使下一阶段的工作看起来准备就绪，即使前置工作、审批、材料或通道条件实际上并未在模型中正确体现。

这一点很重要，因为数据日期是实际绩效与预测之间的边界。如果活动在没有逻辑的情况下聚集于该边界，预测可能变得不可靠。浮时值可能被扭曲，最长路径可能不清晰，项目团队可能对进度计划实际告诉他们的内容失去信心。

对于 Primavera P6 用户而言，该度量指标有助于提升更新纪律、增强 CPM 逻辑，并使进度质量审查更具可论证性。它帮助计划工程师确认工作的启动是出于真实的进度原因，而不仅仅是因为软件将未驱动的活动移动到了当前的更新日期。
