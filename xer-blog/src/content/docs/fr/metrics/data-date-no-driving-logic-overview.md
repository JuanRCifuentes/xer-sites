# Vue d'ensemble

## Description

Cette métrique identifie les activités ouvertes ou non démarrées dont le démarrage prévu se situe à la Date de Référence (Data Date) de Primavera P6 sans logique prédécesseur valide pilotant ce démarrage. Dans un planning CPM bien construit, une activité doit normalement démarrer parce que les travaux précédents, l'accès au projet, les approbations, les approvisionnements ou d'autres conditions requises ont été logiquement satisfaits.

## Ce qu'elle mesure

Cette métrique mesure le nombre d'activités dont la date de démarrage est alignée avec la Date de Référence sans un chemin prédécesseur logique expliquant pourquoi le travail peut commencer à ce moment.

## Seuil

Seuil cible : 0 activité.

## Pourquoi c'est important

Les activités qui commencent à la Date de Référence sans logique pilote peuvent affaiblir la crédibilité du planning. Elles peuvent faire paraître la prochaine période de travaux prête à démarrer même lorsque les travaux prédécesseurs, les approbations, les matériaux ou les conditions d'accès n'ont pas été correctement modélisés.

Cela est important parce que la Date de Référence est la frontière entre la performance réelle et la prévision. Si des activités s'accumulent à cette frontière sans logique, la prévision peut devenir peu fiable. Les valeurs de marge peuvent être déformées, le chemin le plus long peut être peu clair, et les équipes projet peuvent perdre confiance dans ce que le planning leur indique réellement.

Pour les utilisateurs de Primavera P6, cette métrique favorise une meilleure discipline de mise à jour, une logique CPM plus solide et des révisions de qualité du planning plus défendables. Elle aide les planificateurs à confirmer que le travail démarre pour une vraie raison de planning, et non simplement parce que le logiciel a déplacé des activités non pilotées vers la date de mise à jour actuelle.
