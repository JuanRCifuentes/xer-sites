This entry exercises every frontmatter variable Starlight supports. Open the file side by side with this page and check each effect:

## Where to look on this page

- **Top of the page** — the announcement bar comes from `banner`.
- **Browser tab / `<h1>`** — `title`.
- **URL** — `/en/blog/frontmatter-demo/` set by `slug`.
- **Sidebar** — the entry reads "Frontmatter demo" (`sidebar.label`) with a green **Demo** pill (`sidebar.badge`), sits first in its group (`sidebar.order`), and opens in a new tab with a hover tooltip (`sidebar.attrs`).
- **Right panel ("On this page")** — only `##` headings are listed because `tableOfContents.maxHeadingLevel` is 2. The `###` heading below is deliberately missing from it.
- **Footer** — "Last updated June 9, 2026" (`lastUpdated`), a customized **Previous** link (`prev`), no **Next** link (`next: false`), and an "Edit page" link pointing to a custom URL (`editUrl`).
- **Page source** — `description` as the meta description and a fake `og:image` tag from `head`.
- **Search** — the page is indexed because `pagefind` is true.

### This h3 is hidden from the table of contents

If `tableOfContents` used the default `maxHeadingLevel: 3`, this heading would appear in the right-hand panel. It doesn't — that's the field working.

## Things you can't see right now

- `draft: false` — flip it to `true`, run a production build, and the page vanishes.
- `template` / `hero` — commented out in the frontmatter. Uncomment them to turn this into a full-width splash landing page with a hero image and buttons (the sidebar disappears in that mode).
- `pagefind: false` — would silently drop the page from search results.

Delete this file whenever you're done exploring.
