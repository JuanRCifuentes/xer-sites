# 개요

## 설명

이 지표는 Primavera P6 데이터 기준일(Data Date)에 시작하도록 예정되어 있지만 그 시작을 구동하는 유효한 선행 로직이 없는 미착수 또는 미시작 활동을 식별합니다. 잘 구성된 CPM 일정에서, 활동은 일반적으로 선행 작업, 프로젝트 접근, 승인, 조달, 또는 기타 필요한 조건이 논리적으로 충족되었기 때문에 시작되어야 합니다.

## 측정 내용

이 지표는 해당 시점에 작업이 시작될 수 있는 이유를 설명하는 논리적 선행 경로 없이 시작 날짜가 데이터 기준일과 일치하는 활동의 수를 측정합니다.

## 임계값

목표 임계값: 활동 수 0.

## 중요한 이유

구동 로직 없이 데이터 기준일에 시작되는 활동은 일정의 신뢰성을 약화시킬 수 있습니다. 선행 작업, 승인, 자재, 또는 접근 조건이 제대로 모델링되지 않은 경우에도 다음 기간의 작업이 시작 준비된 것처럼 보일 수 있습니다.

데이터 기준일이 실제 실적과 예측 사이의 경계이기 때문에 이것이 중요합니다. 활동이 로직 없이 그 경계에 집중되면, 예측이 신뢰할 수 없게 될 수 있습니다. 여유 시간 값이 왜곡되고, 최장 경로가 불명확해지며, 프로젝트 팀이 일정이 실제로 무엇을 말하는지에 대한 신뢰를 잃을 수 있습니다.

Primavera P6 사용자에게, 이 지표는 더 나은 업데이트 규율, 더 강력한 CPM 로직, 더 방어 가능한 일정 품질 검토를 지원합니다. 플래너가 작업이 단순히 소프트웨어가 구동되지 않은 활동을 현재 업데이트 날짜로 이동시켰기 때문이 아니라 실제 일정 이유로 시작되고 있음을 확인하는 데 도움을 줍니다.
