# 概要

## 説明

このメトリクスは、Primavera P6のデータ日付（Data Date）に開始するようにスケジュールされているが、その開始を左右する有効な先行ロジックを持たない、オープンまたは未開始の作業を識別します。適切に構築されたCPMスケジュールでは、作業は通常、先行作業、プロジェクトアクセス、承認、調達、またはその他の必要条件が論理的に満たされているために開始すべきです。

## 測定内容

このメトリクスは、その時点で作業が開始できる理由を説明する論理的な先行パスなしに、データ日付に揃った開始日を持つ作業の数を測定します。

## 閾値

目標閾値：0作業。

## なぜ重要か

ドライビングロジックなしにデータ日付に開始する作業は、スケジュールの信頼性を低下させる可能性があります。先行作業、承認、資材、またはアクセス条件が適切にモデル化されていない場合でも、次の期間の作業が開始準備完了に見えてしまうかもしれません。

データ日付は実際の実績と予測の境界であるため、これは重要です。作業がロジックなしにその境界に集まると、予測は信頼性を失う可能性があります。フロート値が歪み、最長パスが不明瞭になり、プロジェクトチームはスケジュールが実際に伝えていることへの信頼を失うかもしれません。

Primavera P6ユーザーにとって、このメトリクスは更新規律の向上、より強固なCPMロジック、より防御可能なスケジュール品質レビューをサポートします。プランナーが、作業が単にソフトウェアがドライブされていない作業を現在の更新日に移動させたからではなく、実際のスケジュール上の理由で開始していることを確認するのに役立ちます。
